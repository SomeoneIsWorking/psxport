# Findings — FMV / CD streaming

## Native port of libcd directory cache — CdNewMedia + CdCacheFile (2026-07-04)
- **symptom:** SBS `MODE=gameplay` (pc_skip=true vs recomp) f0 divergence 1273 B — 8 B at `0x800AC2D4..0x800AC2DC` (libcd cached-dir-idx + media cookie), ~500 B at `0x80102D68+` (path-table cache), ~500 B at `0x80102768+` (file-entry cache for last-cached dir), plus 2 KB of dir-sector scratch at `0x80104368+`.
- **status:** fixed — libcd cache region f0 zero-diff; remaining 522 B at f0 are pre-existing Slip #6 (task-fiber cadence), unrelated.
- **cause:** `Engine::startBinStage` (pc_skip's collapsed STAGE-0 native entry) uses `resolve_via_iso9660` (fast native disc lookup) and bypasses libcd entirely. The substrate on B enters the libcd chain via `FUN_800499E8 → CdSearchFile (FUN_8008B8F0) → CdNewMedia (FUN_8008BBE8) + CdCacheFile (FUN_8008BF50)` for each file resolve, leaving the in-memory libcd cache populated (path table @ `0x80102D68`, file entries @ `0x80102768`, current-dir sentinel @ `0x800AC2D4`, media cookie mirror @ `0x800AC2D8`, last-sector scratch @ `0x80104368`). None of this state is read by any native code today (grep confirmed), but any post-boot substrate dispatch into libcd — plus SBS's no-residuals gate — needs A to match B byte-for-byte.
- **fix:** native ports `cdlibcd_new_media(c)` + `cdlibcd_cache_file(c, dir_idx)` in `game/scene/engine_stage.cpp` — read the PVD + L-path-table + directory sector via `disc_read_sector`, walk ISO9660 records, populate the same guest RAM the substrate writes. Also mirrors each sector into the guest scratch buffer at `0x80104368` (via `cdlibcd_read_into_scratch`). Wired into `startBinStage` as `cdlibcd_new_media` → set `0x800AC2D8 = *(0x800ABFD0)` (CdSearchFile's post-newmedia cookie stamp) → `cdlibcd_cache_file(2)` → `cdlibcd_cache_file(3)` (matches substrate's boot chain leaving cache on \CD subdir, verified via SBS wwatch on `0x800AC2D4` — writes 0→2→3 from `ra=0x8008BDA4` (CdNewMedia) then `ra=0x8008C0D0` (CdCacheFile) × 2).
- **refs:** `game/scene/engine_stage.cpp` cdlibcd_new_media / cdlibcd_cache_file (Ghidra decomp scratch: FUN_8008BBE8 + FUN_8008BF50 + FUN_8008B8F0 CdSearchFile). SBS was 1273 B f0 diff → 522 B after; the remaining bytes are all in `0x801FE0xx..0x801FF1FF` (task/stack region — Slip #6 in docs/findings/sbs.md, task-fiber spawn cadence, unrelated to libcd).

## SBS both-mode freezes on the intro OP.STR movie
- **symptom:** `PSXPORT_SBS_MODE=both ./run.sh` freezes after `[fmv] MOVIE/OP.STR -> LBA 152238`, spamming the game's own `time out in strNext()` (seconds apart); window stops updating. PSXPORT_NO_FMV=1 does NOT help.
- **status:** fixed 5a61c3d
- **cause:** OP.STR is owned by the native FMV player (already g_sbs-skipped, native_fmv.cpp:677 `if (g_sbs) return 0` — so core A is fine). Core B (PSX) runs the guest demo machine; its STR streamer strNext (FUN_8010755c) waits for CD-streamed STR sectors NEVER fed on the interp path, busy-polling StGetNext (FUN_8008d030) ~2000x2000 = ~4M interpreted times per attract cycle — a non-yielding multi-second spin that stalls the SBS lockstep.
- **fix:** sbs.cpp step_core: while a core is in the DEMO stage (0x801fe00c==0x801062E4) and its demo SM[0x48]==1 (intro-FMV sub-state), set the game's OWN skip flag DAT_1f80019d — the demo prologue forces the teardown path, skipping the async STR streaming synchronously. Confined to DEMO stage so the GAME-stage cutscene is never skipped.
- **refs:** journal later-253; sbs.cpp step_core; DEMO.bin.asm FUN_8010755c/FUN_801075f8

## CdSync (FUN_8008a6ec) is already HLE'd to report completion
- **symptom:** worry that a `do { FUN_80089e1c(9,0,0)=CdControlB(CdlPause) } while(==0)` teardown loop (or any CdControlB-blocking) will spin forever under the no-IRQ runtime.
- **status:** known-good (do not re-investigate)
- **cause:** CdControlB blocks on CdSync (FUN_8008a6ec), which busy-loops on the CD-IRQ status (never set here).
- **fix:** Already handled — cd_override.cpp:397 platform_hle_register(0x8008A6EC, ov_cd_sync) sets v0=2 (CdlComplete). So CdControlB-blocking commands COMPLETE. The platform-HLE table is consulted on the interpreter path (interp.cpp coro_native_call) for every jal; runtime is interpreter-only (dispatch.cpp rec_func_index always -1). Faking StGetNext frames is a MINEFIELD (re-enters the decode-wait spin FUN_80106f80 case 6 `do{}while(_DAT_1f800034==0)` and corrupts SM[0x4a]); use the demo skip flag instead.
- **refs:** cd_override.cpp:397 ov_cd_sync; sync_overrides.cpp; interp.cpp coro_native_call
